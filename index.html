¿Cuál tipo de búsqueda prefieres?

Estamos diseñando la pantalla de búsqueda de clientes que usarán los vendedores en tienda. Tenemos dos propuestas — pruébalas ambas y dinos cuál te funciona mejor.

Tiempo estimado

probar + votar
Pruébalas: escribe en la búsqueda de la Opción A, luego en la Opción B. Intenta buscar por nombre, número de documento o teléfono. Luego vota abajo.
A

Búsqueda unificada inteligente

Un solo campo · auto-detecta el tipo de búsqueda

Pros

Fastest path for advisors who know one field
No need to switch modes — type and search
Fuzzy matching handles typos and accents
Recent customers shown by default (zero typing)
Post-search filter + sort to refine results

Cons

Ambiguity if customer name = digits (rare)
Without explicit fields, harder for novice users to know what they can search

How it disambiguates

— Pure digits ≥ 4 chars → searches DPI, TRN, phone, account simultaneously
— Contains CRD- → account number
— Words → name (with fuzzy match)
— Empty → recent customers
↑ Prueba: escribe un nombre, un número o "CRD-2026"
B

Criterios explícitos con pestañas

Eliges el criterio primero (Documento / Nombre / Tel / Cuenta)
Document
Name
Phone
Account

Pros

No ambiguity — advisor controls what's searched
Easier to learn for new advisors
Audit trail: what criterion was used
Recent customers always visible (any tab)
Post-search filter + sort to refine results

Cons

One extra click before typing
Less forgiving if advisor picks wrong tab
No global "find anything" capability
↑ Prueba: elige una pestaña y haz una búsqueda
